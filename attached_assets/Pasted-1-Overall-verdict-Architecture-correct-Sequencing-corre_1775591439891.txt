2. Critical fixes before you build
Issue 1 — similarity normalization (MOST IMPORTANT)

You said:

similarity — cosine distance from pgvector

⚠️ Problem:

pgvector <=> returns distance, not similarity
distance = lower is better
your formula assumes higher is better
Fix

Convert:

similarity = 1 - cosine_distance

Then ensure:

similarity ∈ [0,1]

👉 If you skip this:

scoring breaks silently
bad objects rank higher
Issue 2 — frequency_bonus design (very important)

You said:

frequency from traces

⚠️ Risk:

early data bias
popular ≠ correct
feedback loop amplifies noise
Fix

Use capped + dampened version:

frequency_bonus = min(0.1, log(1 + count) * 0.03)

👉 Prevents runaway reinforcement

Issue 3 — source_weight definition (clarify this)

You defined:

number of chunks contributing to object

⚠️ That’s weak proxy

Better version
source_weight =
  (# distinct documents supporting object)

Not:

chunks
fragments

👉 You want cross-source agreement, not chunk density

3. Dedup layer — refine slightly

Better dedup flow

4. Diversity layer — refine logic

👉 Don’t reduce strong signals unnecessarily

5. Scoring formula — final recommended version

Use this:

final_score =
  (0.6 * similarity)
+ (0.2 * object_confidence)
+ (0.1 * source_weight)
+ (0.05 * frequency_bonus)
+ (0.05 * canonical_boost)
Why tweak:
frequency reduced (safer)
canonical included explicitly
keeps balance clean
