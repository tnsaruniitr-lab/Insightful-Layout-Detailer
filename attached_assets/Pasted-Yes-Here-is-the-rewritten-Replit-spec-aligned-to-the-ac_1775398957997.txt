Sieve — Phase 1 Spec
Marketing Brain + Brand Mapper

Goal: Build the intelligence layer first, then apply it to a specific brand.

This is not a generic audit tool and not yet a full autonomous marketing system.

Phase 1 should create a marketing brain that:

learns structured SEO / GEO / AEO intelligence from documents and trusted sources
stores that intelligence as reusable principles, playbooks, rules, and anti-patterns
maps that intelligence to a specific brand profile
answers brand-specific questions
suggests where strategy should start

Later phases can add:

weekly planning outputs
execution workflows
skills
learning loops
autonomous agents
1. Product Definition
What the product is

Sieve is a marketing intelligence operating system.

At its core, it has 3 layers:

Layer 1 — Knowledge Brain

Stores reusable intelligence about:

SEO
GEO / AI search visibility
AEO / answer optimization
content strategy
entity strategy
comparison / alternatives strategy

prioritization heuristics

Stores one brand’s specifics:

which knowledge is relevant

Uses brain + brand context to generate:

answers to questions

strategic priorities

7. Knowledge Ingestion Pipeline
Goal

Turn raw documents into structured marketing intelligence.

Pipeline steps
Step 1 — ingest source

Input:

uploaded PDF/doc/text/markdown
optional web import later

Output:

document record
raw stored file
extracted text
Step 2 — chunk and embed

Split into chunks with metadata:

source
title
topic
domain
author
trust level

Store in document_chunks.

Step 3 — classify

Classify chunks into categories such as:

SEO fundamentals
GEO / AI visibility
answer structure
entity strategy
comparison content
bottom-funnel content
content quality
anti-patterns
Step 4 — distill intelligence

Use extraction workflows to convert chunks into structured objects:

principle
rule
playbook
anti-pattern
example
Step 5 — deduplicate and merge

If similar principles or playbooks already exist:

merge
increase source count
update confidence
preserve provenance
Step 6 — mark as canonical or candidate

Phase 1 should allow:

canonical = approved brain knowledge
candidate = extracted but not yet trusted enough

This is important for future self-learning.

8. Brain Logic
The brain should not be “just RAG”

It should combine:

semantic retrieval from source chunks

output formatting logic
Retrieval priority

When answering a question:

relevant anti-patterns

This keeps answers grounded in structured intelligence first.

For:

Rule

Do not let the model invent canonical intelligence without source basis.
Every canonical object should include source references.

15. Trust and Quality Rules
Source hierarchy

Rank source trust roughly as:

official / primary docs
high-quality expert sources
internal strategy docs
selected blogs
weaker community commentary
Canonicalization rule

A principle or playbook should become canonical only if:

supported by enough strong evidence
not contradicted by higher-trust sources
expressed clearly enough to be reusable

Otherwise keep it as candidate.

Answering rule

Every answer should separate:

known principle
brand inference
uncertainty
missing data
16. Scalable Architecture for Later Skills and Self-Learning

This Phase 1 architecture should support later expansion without rewrite.

16.1 Skills architecture later

Later, each skill can sit on top of the same brain:

alternatives-page skill
GEO content skill
refresh-page skill
internal-linking skill
entity-consistency skill
comparison-page skill

Each skill should consume:

playbooks
rules
brand context
optional structured data

So the brain becomes the shared intelligence substrate.

16.2 Self-learning later

Do not implement uncontrolled self-learning now.
But design for it now.

Future learning loop:

ingest new docs or web sources
extract candidate principles / playbooks
compare with current canon
propose merges / updates
require approval or confidence threshold
update canon with provenance
16.3 Data-driven strategy later

When Phase 2 structured uploads mature:

rankings
traffic
GEO snapshots
competitor snapshots

Then the strategy engine can combine:

canonical brain knowledge
brand context
observed performance data

That is when audits and 14-day plans become much stronger.
