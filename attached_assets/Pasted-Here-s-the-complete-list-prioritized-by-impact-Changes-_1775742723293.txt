Change 2 — Anti-pattern prompt: add contrastive framing patterns
Line 612, extend the derivable don'ts bullet to:
- Derivable don'ts: if the content says "avoid X", "do not use Y", "Z will be ignored",
  "A is not supported", "don't block with B" — extract as anti-pattern
- Contrastive framing: "rewards X rather than Y" or "designed for X, not Y" → Y is an anti-pattern
- Negated evaluation: "does not evaluate/support/apply to X" → assuming it does = anti-pattern
- Outcome framing: "content that fails to X is likely to perform poorly" → failing X = anti-pattern
- Implicit contrast: any sentence where a negative outcome is implied by the absence of a positive quality
Change 3 — Playbook prompt: add prose procedure recognition
After line 513, add:
- A sentence where a single actor is given 3+ sequential actions using infinitive verbs
  (e.g. "site owners should review...ensure...add...apply") is a playbook even without
  numbered formatting — extract the actions as ordered steps
- Advisory checklists ("owners should: [comma-separated actions]") are playbooks
- Look for the pattern: actor + "should" + 3 or more verbs = playbook candidate
Change 4 — Rules prompt: add specificity test for if_condition
Currently the specificity gate rejects vague conditions but doesn't guide improvement. Add after line 423:
- If an if_condition seems vague, sharpen it using surrounding context rather than skipping it
- A good if_condition names a specific observable state: "if using 301 redirect for test variant"
  not "if running a test"
Change 5 — Principles prompt: add rule/principle disambiguation
After line 347, add:
- If a concept has a clear IF/THEN structure (a trigger and an action), it is a RULE not a principle
- Do not extract "X pages are not eligible for Y feature" as a principle — it is a rule
  with if_condition: "if page is classified as X" and then_logic: "then not eligible for Y"
- A principle describes a universal truth; a rule describes a conditional action
