The Drizzle schema has contested, conflict_pair_id, negation_embedding_vector defined — but they're not in Supabase. I queried directly:

Issue 2 — Enrichment body update is incomplete (line 779)
typescriptconst refs = [...new Set([...JSON.parse(row.source_refs_json || "[]"), ...JSON.parse("[]")])]
The second JSON.parse("[]") is hardcoded empty — it should be JSON.parse(candidate.sourceRefsJson || "[]"). The enrichment merge correctly updates source_refs_json on the existing object but isn't adding the new document's source refs. The candidate's document ID gets lost. The source_count also isn't being bumped.
Issue 3 — canonicalBoost weight in scoring formula is double-applied
In scoring.ts line 154–160:
typescriptconst canonicalBoost = c.isCanonical ? 0.05 : 0;
const finalScore = ... + 0.05 * canonicalBoost;
When isCanonical = true, canonicalBoost = 0.05, then 0.05 * 0.05 = 0.0025. The actual canonical boost to the final score is 0.0025, not 0.05. The signal is effectively dormant still. Should be either:

const canonicalBoost = c.isCanonical ? 1 : 0 (binary flag, weight applied in formula), or
0.05 * (c.isCanonical ? 1 : 0) directly in the formula

Issue 4 — Backfill script missing
Task 2 in the spec ("write a one-time migration script that runs the same promotion check against all existing objects") has no corresponding file in the repo. With 400+ existing objects that may already meet the criteria, this is the most immediate way to activate canonicalBoost in production. Without it, canonical promotion only fires on future ingests.
Issue 5 — buildAuthorityMap uses trust_level not authority_tier
The function queries trust_level from documents — which is the existing high/medium/low field, not a new authority_tier field. This is correct for now (step 2 authority tier doesn't exist yet) but the TIER_WEIGHTS are labelled high/medium/low rather than tier-1/tier-2/tier-3. When step 2 ships and adds authority_tier, you'll need to decide whether to keep using trust_level as a proxy or migrate to the new field. Worth flagging now so it's an explicit decision rather than an accidental one.
